# 超级赞 Super-Z

一个群众帮助群众的平台,旨在提高国内群众挣到的钱,向外卖平台等第三方黑心平台宣战。

> **超级赞是「群众帮群众」计划的第一战。** 目标不只是外卖:凡是高抽成、不透明的行业,这套「低抽成 + 账目公开 + 代码开源」的模式都成立。外卖打穿之后,再打下一个。
>
> ⭐ 如果你认同「外卖可以不是这样」,给个 star 就是最简单的支持。也欢迎提 issue / PR / 把它部署到你的城市——或者来帮我们打下一个战场。

**低抽成外卖平台**:商家抽成 5%(行业平台 20%+),用户端 / 商家端 / 骑手端三端 + 一个后端。

## 平台三原则(产品决策以此为纲)

1. **商家只抽 5%** —— 菜价里没有平台税,点单页明示。**5% 是上限,不是目标:哪天 3% 能活,就降到 3%(这行写在这里当立字据)**
2. **配送费 100% 归骑手** —— 按距离计价,平台分文不取,提现零手续费
3. **账目三方透明** —— 每一单"钱去哪了"(商家实收/骑手所得/平台留存)对用户公开可查

透明本身就是与吸血平台竞争的武器。5% 是唯一收入(不卖数据、不竞价排名、不收推广费、不抽配送费),平台自身收支定期公示,盈余不分红——优先骑手保障、商家扶持和降费率。

## 架构

```
super-z/
├── apps/
│   ├── user_app/        # 用户端(Flutter):浏览商家、下单、追踪订单
│   ├── merchant_app/    # 商家端(Flutter):接单、出餐、营业开关
│   └── rider_app/       # 骑手端(Flutter):抢单、取餐、送达、位置上报
├── packages/
│   └── shared/          # 三端共享:数据模型、API 客户端、登录页
├── server/              # 后端(FastAPI + PostgreSQL/PostGIS + Redis)
│   └── app/
│       ├── state_machine.py   # 订单状态机(核心,所有状态变更的唯一入口)
│       ├── models.py          # 数据模型(users/merchants/dishes/orders/order_events)
│       └── routers/           # auth / merchants / orders / riders
└── docker-compose.yml
```

**订单状态流**:待支付 → 待接单 → 制作中 → 待取餐 → 配送中 → 已送达 → 已完成
(骑手在「制作中/待取餐」阶段抢单;每次流转写入 order_events 审计表)

## 快速开始

### 1. 后端(5 分钟)

```bash
# 起 PostgreSQL(PostGIS)和 Redis
docker compose up -d db redis

# 安装依赖并启动 API
cd server
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload          # http://localhost:8000/docs 看接口文档
# 如果 8000 被占用(比如本机还跑着别的项目),加 --port 8010,
# Flutter 端用 --dart-define=SUPERZ_API=http://127.0.0.1:8010 对应

# 另开一个终端,灌演示数据
cd server && source .venv/bin/activate && python -m scripts.seed
```

或者全容器化:`docker compose up -d --build`(API 也跑在容器里)。

## 关键设计决策

- **金额一律用「分」存整数**,杜绝浮点误差
- **状态机集中管控**(`state_machine.py`):任何非法流转直接 409,每次变更写审计表
- **防超卖**:扣库存用条件 UPDATE(`stock >= quantity`),数据库层面保证并发安全
- **防重复支付**:支付接口幂等,重复回调直接返回当前状态
- **抢单防冲突**:条件 UPDATE(`rider_id IS NULL`),同一单只有一个骑手能抢到
- **骑手位置只写 Redis**(5 秒一报),不给 PostgreSQL 压力
- **模拟支付可平滑替换**:接微信支付时把 `pay/mock` 的逻辑搬进支付回调即可,幂等结构不变
