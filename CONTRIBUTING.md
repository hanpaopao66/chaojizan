# 参与 Super-Z

感谢你的兴趣。这个项目的目标是证明:平台可以不吸血也活得下去。

## 参与方式

- **报告问题**:开 Issue,说清复现步骤;涉及资金账目的问题请标 `money` 标签,最高优先级处理
- **提交代码**:Fork → 分支 → PR;PR 说明里写清楚动机和验证方式
- **安全漏洞**:不要公开开 Issue,请通过 App 内客服工单或维护者邮箱私下报告

## 代码约定

- **金额永远以「分」(int) 传输和存储**,展示层才转元
- **账本只追加**:收入流水表不允许 UPDATE/DELETE,冲账 = 追加负数行
- **改动资金逻辑必须同步改审计**:`server/app/services/audit.py` 的恒等式
  是账本的守夜人,任何定价/结算/退款改动都要让恒等式继续成立,并补 e2e
- **状态变更只走状态机**:`server/app/state_machine.py` 是唯一入口
- 数据库结构变更走 Alembic(`alembic revision --autogenerate`),禁止手写 ALTER
- 服务端跑 `make test` 全绿、Flutter 包 `flutter analyze` 零问题才提 PR

## 三原则是底线

商家 6% 佣金、配送费 100% 归骑手、账目三方透明——
动摇三原则的 PR 不会被合并,无论技术上多么优雅。

## 把它带到你的城市

AGPL-3.0 允许你部署自己的实例(修改必须开源,品牌请换成你自己的)。
如果你真的这么做了,欢迎回来开 Issue 交流运营经验——群众帮助群众。
