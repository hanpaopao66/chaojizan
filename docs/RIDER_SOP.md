# 骑手交通事故处理 SOP(客服照此执行)

> 原则:人先安全,钱后面算。事故上报的技术动作(在途单自动释放/转仲裁、
> 加急工单、管理员推送)系统已自动完成,本文是**人工跟进**的动作清单。

## 收到事故推送后(5 分钟内)

1. **立即电话回访骑手**(管理后台「运力」面板事故卡有电话):
   - 先问伤情:需要救护车吗?已报警了吗?(受伤未叫车 → 替骑手拨 120)
   - 记录:时间/地点/伤情/是否涉第三方/车辆状况。
2. 管理后台把事故标记为「跟进中」,备注回访结果。

## 跟进中(当天)

3. **保险/保障金**:
   - 事故日已投保(insured):指引骑手按保单号报案理赔,平台协助提供材料;
   - 登记模式(registered):走保障金池先行赔付——医疗票据实报实销,
     金额与依据在事故留痕里写清(保障金池收支进公开账本)。
4. **在途订单**:系统已自动处理(未取餐回池 / 已取餐转配送异常仲裁);
   仲裁时一律按骑手无责处理(fault=platform 口径,用户全额退款平台承担)。
5. 涉第三方的:提醒骑手保留交警责任认定书,平台不替任何一方定责。

## 结案

6. 伤愈复跑 / 赔付到位 / 理赔完结后,标记「已结案」,备注:
   赔付金额与出处(保险/保障金池)、责任认定结果、骑手状态。
7. 每季度汇总事故数据(次数/类型/路段),用于安全培训题库更新与装备采买。

## 红线

- 不催促伤者继续接单;结案前该骑手的考核数据(转单/时长)全部豁免。
- 不垫付≠不赔付:保障金池的用途就是先行赔付,别让骑手垫医药费等理赔。

# 骑手紧急求助(SOS)SOP

> SOS 是"进行中的危险"(抢劫/骚扰/被困/突发不适),比事故上报更急。
> 系统动作:红色加急工单(带位置与在途单数)+ 管理员推送 + 紧急联系人
> 短信(未接短信服务时工单会标注"需人工联系紧急联系人")。

1. **5 分钟内电话回访**,先确认人身安全;打不通立即再打,连续 3 次
   打不通且有定位的,视情况代报警(110)并联系紧急联系人。
2. 确认安全前**每 10 分钟跟进一次**,后台标记「跟进中」逐次留痕。
3. 在途订单**不自动释放**(SOS 误触率高):与骑手确认无法继续配送后,
   由客服走管理改派(未取餐)/配送异常仲裁(已取餐),一律骑手无责。
4. 误触:骑手 2 分钟内可自助撤销;超窗的误触在回访电话里确认后结案。
5. 结案备注:事发类型/处置结果/是否报警/联系人是否触达;
   数据季度复盘用于夜间运营策略(如高危时段/路段提醒)。
