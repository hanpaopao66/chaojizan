# Super-Z 商业化完善度评审与功能设计(执行提示词)

> 用法:对 Claude 说「按 docs/prompts/commercial-review.md 开干」即可。
> 可加参数:「只做评审不实现」「只做 P0」「跳过第 X 维度」。

## 角色与目标

你是 Super-Z 的产品负责人兼主程。以一线商业外卖/本地生活产品(美团外卖、
饿了么、Grab、大众点评)的完成度为标尺,系统评审 Super-Z 的商业化短板,
产出可落地的功能设计与优化方案,并按优先级逐项实现、验证、提交。

## 项目背景(执行前必读)

- 愿景:劳动者互助平台,向吸血中介行业宣战。三原则是产品决策的上位约束:
  **商家只抽 6% / 配送费 100% 归骑手 / 账目三方透明**
- 已上线业务:外卖(完整)、团购券(核销收 3%);金刚区占位:打车/跑腿/家政/维修/货运/零工
- 工程现状:M1~M6 全部收官(听单保活/生产加固/资金闭环/上架合规/精细运营/开源流水线),
  22+ 个 e2e 模块,审计守夜人 8 条资金恒等式,品牌体系(炉火橙/账目绿/Logo/插画/微动效)
- 必读文件:docs/ROADMAP.md(里程碑与勾选)、docs/BRAND.md(品牌与 UI 规范)、
  docs/ACCEPTANCE.md(真机验收)、server/app/services/audit.py(资金恒等式)、
  CONTRIBUTING.md(账本铁律)
- 技术栈:FastAPI+PostgreSQL(PostGIS)+Redis;Flutter 三端 + shared 包;Alembic 迁移

## 审视维度(逐项给出:现状证据 → 差距 → 值不值得做)

逐条读代码取证,不凭印象。每条标注:✅完善 / 🟡可用但粗糙 / ❌缺失。

1. **转化链路**:首页→店铺页→购物车→支付,每一步的流失点。
   重点检查:商家卡是否给足决策信息(招牌菜/评分/时效)、搜索(热词/历史/纠错)、
   菜品图覆盖率的引导、店铺页装修(相册/公告)、结算页摩擦
2. **复购与留存**:再来一单(已有,可再深化?)、收藏的触达、
   推送召回策略(JPush 就位后)、会员/储值——注意资金池红线,储值大概率不做
3. **信任与口碑**:评价体系深度(图片评价/标签化/追评/商家回复的触达)、
   新店冷启动的信任构建(三原则如何在店铺页变成转化力)
4. **履约体验**:订单页实时地图/骑手位置(骑手端已有上报)、预计送达倒计时、
   超时的主动安抚与补偿规则
5. **商家侧商业化**:数据看板(经营诊断:哪个菜卖得好/差评归因)、
   活动工具(限时折扣/第二份半价——满减已有)、出餐提效
6. **客服与纠纷**:退款时效的可视化承诺、申诉链路完整性、工单响应体验
7. **增长**:分享店铺/菜品/团购券(微信生态,合规)、邀请有礼(成本透明化)、
   首单立减(已有)之外的拉新
8. **团购深化**:套餐券(内容型券)、券详情图文、门店相册、到店导航
9. **运营地基**:公告/弹窗的服务端配置化、简单灰度开关、
   自建埋点(必须写进隐私政策,不接第三方统计)
10. **收尾质量**:空态/异常态全覆盖检查、弱网体验、无障碍基线

## 硬约束(违反即返工)

1. 三原则不可破;让利类功能必须在账本上有名有姓(参考满减/首单立减的实现)
2. 任何资金相关改动:同步 audit.py 恒等式 + 新增 e2e,`make test` 全绿才算完成
3. 不做焦虑式营销、不做诱导储值(资金池红线)、不做竞价排名
4. UI 严格遵守 BRAND.md(炉火橙=行动、账目绿=钱、不放口号横幅);
   新页面要有空态插画与微动效,与现有体系一致
5. 埋点如做,自建方案且同步更新隐私政策全文(shared/legal.dart + 服务端网页版)
6. 数据库结构变更走 Alembic;客户端 `flutter analyze` 零问题;shared 包 widget test 通过
7. 每个功能独立 commit(不 push),提交信息说清动机与验证方式

## 执行方式

1. **评审**:产出 docs/COMMERCIAL-REVIEW.md——
   表格:维度 / 现状(代码证据) / 差距 / 方案一句话 / 优先级 P0-P2 / 预估规模(S/M/L)。
   优先级依据:对"用户真实感受到好用"的贡献 ÷ 实现成本,合规风险一票否决
2. **确认**:P0 清单向我复述一遍(一段话即可)后直接开干;P1/P2 等我点头
3. **实现**:按优先级逐项做,每项:实现 → e2e/analyze/widget test → 独立 commit;
   需要真机才能验证的项,写进 docs/ACCEPTANCE.md 对应板块
4. **收尾**:更新 ROADMAP.md;给我一份"这轮做了什么/验证结果/下轮建议"的总结
