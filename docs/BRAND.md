# Super-Z 品牌规范 v2

> v2 变更:统一「传播层」与「产品层」双层色板;Logo 更换为定稿的「赞手 + 透明账目」;
> 其余布局/插画/动效原则保持 v1 不变。
> 代码里的唯一来源不变:`packages/shared/lib/src/brand.dart`。

## 品牌人格

劳动者互助平台:**真诚、利落、有烟火气**。
不端着(不是精英范),不喊口号(信任靠体验),不装可爱(工具要利落)。
对外传播可以有立场和火气(第一战/打穿一个再打下一个),产品内保持克制——
**App 里不贴战斗口号,让用户在账单里自己撞见证据。**

## 品牌色:双层体系

### 传播层(海报/官网 hero/背景图/传单——对外喊话时用)

| 色 | 值 | 用途 |
|---|---|---|
| **品牌渐变** | `#FF7A45 → #E1251B`(左上→右下) | Logo 底、海报背景、官网 hero |
| **超级红** | `#E1251B` | 传播物料的强调色、横版 logo 副标 |
| **金黄** | `#FFD34D` | 配送费/烟火气;海报高亮文字、logo 袖口 |
| 墨色 | `#231815` | 传播物料深色文字/深色底 |

### 产品层(三端 App 内——干活时用)

| 色 | 值 | 用途 | 禁忌 |
|---|---|---|---|
| **炉火橙** | `#FF5A1F` | 行动色:按钮、价格、选中态 | 大面积铺橙;当背景色 |
| **账目绿** | `#0E8A5F` | 钱的正向语义:商家实收、骑手所得、余额、到账 | 用在与钱无关的地方 |
| 促销琥珀 | `#B25E09` | 满减/折扣标签 | 抢过行动色的视觉权重 |
| 暖灰底 | `#F6F6F8` | 页面背景(亮色模式) | |

两层的关系:炉火橙是品牌渐变的"可用色化"——渐变作按钮会显脏,单色 `#FF5A1F`
既在渐变色域内,又满足行动色的对比度需求。**App 内出现品牌渐变的地方只有两处:
Logo 本身、以及「平台承诺卡」这类立场表达组件。** 其余一律用产品层单色。

选色理由:橙 = 食欲与烟火气,与美团黄/饿了么蓝直接区隔;
绿单独留给钱,是"这笔账踏实"的视觉语言——透明不靠说,靠每一处钱的颜色一致。

## Logo(v2 定稿:赞手 + 透明账目)

**竖起的大拇指 = 「赞」**(对应品牌名,群众互相帮衬的姿态);
**手掌三道横线 = 指缝,也是摊开的账单**(商家/骑手/平台三方分账一目了然);
**金黄袖口 = 配送费与烟火气**。渐变圆角底,圆角率 116/512 ≈ 22.7%。

- 矢量源:`marketing/brand/icon_A.svg`(viewBox 512)
- App 内展示:`BrandLogo` 组件(CustomPaint,几何参数与 SVG 一致,两处必须同步)
- App 图标/头像:`marketing/brand/icon_A_1024.png` 一张通用
- 横版 logo(官网顶栏/海报):`marketing/brand/logo_horizontal_light|dark.svg`
- **图标内不加文字**:图标旁永远有名字;文字场景一律用横版 logo
- 旧 logo(闪电 Z + 碗弧)废弃,存量出现处见到即换

## 布局原则(商业化 ≠ 花哨)

1. **首屏第一交互给搜索**:大搜索框直接可见,不藏在图标里
2. **不放口号横幅**:三原则的正确表达是——
   订单详情里可查的「钱去哪了」卡、店铺页的「本店仅被抽成 5%」实数、
   券详情里的「未使用随时全额退」按钮。让用户撞见,而不是被告知
3. **卡片克制**:圆角 14、无阴影(elevation 0)、留白撑质感
4. **促销不抢行动**:满减标签用琥珀描边小签,主按钮永远是最橙的那个

## 字体与字号(补充)

- 中文:系统字体(iOS PingFang SC / Android 系统默认);对外物料用思源黑体(Noto Sans CJK)
- 层级参考:大标题 22/加粗、卡片标题 17/加粗、正文 15、辅助 13、金额用等宽数字(tabular figures)
- **金额永远比它的说明文字大一级**,钱是这个产品的主角

## 插画(brand_art.dart)

空状态不用灰图标,用有温度的小画。统一手法:暖色大圆底 + 圆头描边图形 +
品牌橙点缀一笔 + 不对称装饰小点。五款:碗(没商家/没菜)、票券(没券)、
单据(没订单)、断线云(没网)、放大镜找碗(搜索无结果)。
`EmptyState` 按图标语义自动匹配,调用方零改动。

## 微动效(ui_bits.dart)

原则:**反馈要快(≤200ms),装饰要省**。动效是确认操作被响应,不是表演。

| 场景 | 动效 | 时长 |
|---|---|---|
| 列表首屏进场 | FadeSlideIn:淡入+上滑,逐项错开 45ms,仅前 8 项 | 320ms |
| tab 切换 | 淡入 + 1.5% 微上滑 | 160ms |
| 空态插画/成功图标 | PopIn:0.6→1 弹性缩放 | 450ms |
| 购物车角标 | 数量变化弹跳(elasticOut) | 350ms |
| 结算合计 | 金额上滚换字 | 200ms |
| 骨架屏 | 透明度呼吸 | 900ms 循环 |
| nav 图标 | 描边→实心切换(M3 自带指示) | 系统 |
