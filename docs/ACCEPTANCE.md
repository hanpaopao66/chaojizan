# 真机验收清单

> 准备:一台 Android 手机(建议再借一台,商家端/用户端各装一台体验更真实)。
> 构建:`cd apps/<app> && flutter build apk --release`,
> 或用 scripts/release_apks.sh 发到 appdist。
> 后端:真机与电脑同一 WiFi,构建时加
> `--dart-define=SUPERZ_API=http://<电脑局域网IP>:8000`;或直接连生产 aikas.com.cn。

## C. 外卖全流程(用户端 + 商家端 + 骑手端)

- [ ] 主页:品牌横幅、满减标签、起送价、四 tab 正常
- [ ] 带规格菜品:点 + 弹规格面板,必选组默认选中,价格实时变
- [ ] 购物车:同菜不同规格分行;未达起送价按钮置灰显示差额
- [ ] 结算页:打包费/满减行、预约送达选择(选 30 分钟内被拒)
- [ ] 下单支付 → 商家接单出餐 → 骑手抢单(有预约标签)→ 送达 → 确认
- [ ] 订单详情「这一单的钱去哪了」:商家实收/骑手所得/平台留存/让利补贴行
- [ ] 售后:申请 → 商家同意 → 显示退款;商家端对账页金额归零(冲账)

## D. 团购全流程

- [ ] 商家端:店铺页 → 团购券管理 → 发布一张券(如 45 购 50)
- [ ] 用户端:团购 tab 出现该券(折扣标签正确)→ 抢购 → 支付
- [ ] 我的-我的券包:券显示二维码 + 数字码
- [ ] 商家端:核销页**扫码**用户的二维码 → 弹「核销成功+本单应收明细」
- [ ] 再扫同一张 → 提示已核销不能重复
- [ ] 另买一张不核销 → 用户端申请退款 → 状态变已退款
- [ ] 商家端下载对账单 CSV(需 Web 或接口),含外卖与团购核销行
